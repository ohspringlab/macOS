DGX Files

DGX files are used for importation and exportation of User Recipes and their Categories, categories that can then be used for any recipes, be they preloaded or user-defined recipes. DGX files serve also to backup User Recipes.

DGX files are text files with DGX Format; under OS X DGX files can have any filename that includes the file extension: '.dgx', e.g. , MyOwnRecipes.dgx.

Under iOS only, there is a specific import filename required, namely:

DG_Recipes_Import.dgx

See onboard help in iHungryMe for further information on Recipe importation and exportation under iOS. See Menu Items for ways to Import and Export DGX files to and from HungryMe

DGX File Format

DGX files are simple text files which can be created in a text editor such as Text Wrangler (Mac) or Notepad (Windows) or in a word processor. A DGX file can contain any number of user recipes. A DGX file can sync recipes between OS X and iOS versions; HungryMe and iHungryMe.There are three requirements for the formation of a recipe in a DGX file.

1. A correctly formatted Recipe Name line, for example: '[[ Apple Pie :: Dessert :: User-Defined]]'

2. A case-sensitive tag indicating the begin of Recipe Directions, namely, 'Directions@'.

3. A case-sensitive tag indicating the begin of Recipe Comments, namely, 'Comments@'.

A Recipe for A Cup of Tea

[[ A Cup of Tea :: Treats :: User-Defined :: Some Other New or Old Category]]

1 cup boiling water

1 tea bag

Directions@

1. Steep tea bag for one minute or more.

Comments@

Enjoy hot tea anytime!

Note: There are three Categories for this recipe. If these Categories pre-exist the recipe, they are used, if not, they are added to the database for use by any existing or future recipes. The amount of white space in the recipe text is unimportant with two exceptions. There may be no white space between the '[[' or ']]' double square brackets (Recipe Name delineators) or the '::' double colons (Category name annunciators).

If you choose to use a pre-existing Category in your DGX file, check its spelling carefully. For example, if you want to use the pre-existing Category 'Treats' and you instead type 'Treat', you would end up with two Categories when you, in fact, want one.

Ensuring Correct Recipe Detail Display

Whether you change line length by resizing a Mac window or by rotating an iOS device, the operation system needs to dynamically wrap the lines of the recipe details, as it sees fit. You ensure that this correct line wrapping is done for recipes by following these rules prior to saving your DGX file:

1. Ensure that each Ingredient is on a single line without embedded linefeed characters.

2. Ensure that each Instruction is on a single line without embedded linefeed characters.

3. Ensure that each Comment paragraph is on a single line without embedded linefeed characters.

Many text editors, for example, Text Wrangler, have single commands which remove embedded linefeeds from a paragraph or any block of text.

The Best Editing Approach

1. Using a text editor, enter the text for the DGX file - wrapped as you see fit.

2. Remove all embedded linefeeds.

3. Save as a DGX file.

Reimporting A DGX File

Retain your import DGX file. Keep adding your recipes to it. Each time you import a DGX file, previously imported recipes are skipped over.

Under OS X only, you can use any filename that includes the '.dgx' extension for your DGX files.
